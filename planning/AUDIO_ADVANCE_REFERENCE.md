# Audio Advance — Reference (from the lead's working spreadsheet)

**Source:** the audio advance lead's (Joe King) live Excel used to advance festival
stages — example: *TNT Bloomington 2026 — MAIN STAGE*. Captured 2026-06-23 as
**reference** for the advance content model (mainly informs **Phase 4 — advance
content** and **Phase 5 — packets**; surfaces a few new feature ideas).

> Status legend for mapping: **✅ covered** by ROADMAP today · **➕ extends** an
> existing item (adds specificity) · **🆕 new** — not yet in the plan, confirm
> whether to include.

This is the **audio** department's advance, but the sheet captures cross-department
info (lighting/video/rigging/pyro/labor) — mostly marked "NO" when not audio's
concern. It embodies a clear philosophy: **most production is a standard festival
package; the advance captures each artist's additions, exceptions, and concerns.**
That maps directly to our **templates** (§6) + **section status** (§5) model.

## Workbook structure (3 tab types)

### A. Per-artist advance tab (editable — one tab per stage; artists across columns)
A header block (event/stage identity) + a long list of grouped fields, one column
per artist. Our model: this is the **Advance** (one per artist), its groups are the
**sections**, and the header block is the **Event/Stage**.

**Header / identity block** (event- or stage-level):
- Producer · Show Name · **Stage Name** · Day 1–4 (dates) · Website · Venue ·
  Address · Location · **Google Maps** link · Published Date · "A Few Words" (blurb)
  → ✅/➕ Event fields. We have name/dates/venue. **🆕 add:** producer, stage name,
  website, maps link, published date, blurb. **Note the per-stage structure** (see
  open question: stage as a layer).

**Per-artist top fields:**
- Performance Time · **Advance Completed?** (the overall done flag) · Artist ·
  band/DJ/solo · **Schedule:** Arrival · Crossload · Load-in · Soundcheck · Backline
  Drop-off · Load-out · Crossload Return · **Set time** · Drive Folder Created ·
  Direct Pay/Settlement · Date of advance
  → ➕ schedule fields map to §5 schedules; **Advance Completed?** maps to our advance
  status; **Drive Folder Created** → Drive integration (§12); **🆕 Direct Pay/Settlement.**

**Sections (grouped):**
- **DOCUMENTS** — Production Rider · Stage Plot · Input List (received? checkboxes) → 🆕
- **CONTACTS** — Production Manager / Tour Manager / Audio / Additional, each with
  Cell + Email → ✅ §11 (contacts), but embedded per-advance here
- **STAFF** — FOH Engineer · Monitor Engineer · Playback · Backline · LD · Programming
  Time · VJ · Content/Visuals · Additional Tech Crew · Total Personnel (who's
  covering each role; fest-provided vs artist-provided) → 🆕
- **TRANSPORTATION** — Semis · Box Truck · Buses · Vans · Trailers · Fest Transpo ·
  Personal Vehicle → ✅/➕ §5 transportation (adds semi/box-truck/van/fest-transpo)
- **POWER** — Audio · Lighting · Video · Pyro · Bus Shore Power (requirements) → 🆕
- **BACKLINE** — Rented/Carried/Shared · Backline List · Backline Notes · Direct
  Pay/Settlement → 🆕
- **RISERS** — Carried/Notes + typed counts (4'x8'x9", 8'x8'x12/18/24/36", 2x8 sled
  w/ wheels, 4x8 sled, stairs, custom, DJ riser w/ wheels) → 🆕
- **AUDIO** — FOH Console · MON Console · Playback specs · Other Tech worlds · Power
  needs · Snake · Sub Snakes · Sub Snake Boxes · Patch Notes · Mics & DI's · Stands &
  XLR · **Stands & XLR – REQUIRED** (typed counts: tall/short tripod/round/straight
  booms, LP claw, Zbar, cab grabber, Manfrotto clamp, mic clip, sandbags, XLR 10/25/50)
  · MON needs · RF · IEM · COM/SHOUT/B · Concerns · Band Additions · Crew notes ·
  Pending Unknowns → 🆕 (the audio department's core content)
- **LIGHTING** — Console · Stage Racks/Dimmers · Snake · Power · Spots · Fixtures → 🆕
- **GAS: PYRO** — CO2 #20/#50 · Nitrogen 60cuft/#200 · Propane #20 → 🆕
- **RIGGING** — Motors · Motor Control · Truss · Spanner → 🆕
- **VIDEO** — Walls · Carts · Control · FOH Cameras · Motors → 🆕
- **LABOR** — Loaders · Hands · Heavy Eq · Riggers (counts) → ➕ §5 stagehand labor

### B. Summary report tab (auto-filled from the advance tabs)
Per-day list of every artist with: **Advance completed?** · **Band Additions and
requests outside of festival package** · **Band Concerns and Pending Items**.
→ This is our **PDF packet** (§7 / Phase 5) and **tracker** (§8 / Phase 6). Confirms
the "additions/concerns/pending" fields should be **structured** so they roll up.

### C. Gear totals / pull-sheet tab (auto-filled — "DO NOT EDIT")
Artists across columns; rows = a **full mic/DI/stand model library** (AKG, Audix,
Beyer, Earthworks, EV, Shure, SE, Sennheiser, Telefunken, Radial DIs, plus the
stands/XLR/CAM-tie-in items). Columns: **On-site Available · Total in Use ·
Available/Shortages** (negative = short, flagged red). Auto-sums each artist's
requested quantities and compares to house stock.
→ 🆕 **Gear inventory + pull sheet + shortage calculator** — a new capability:
maintain house stock, advances contribute per-item quantities, the event rolls up
totals and surfaces shortages. High operational value; non-trivial.

## New concepts to decide (raised with the user 2026-06-23)
1. **Advance content breadth** — full multi-department taxonomy vs audio-first vs
   department-configurable (sections appear per the event's departments).
2. **Gear inventory / pull-sheet / shortage engine** — include (own phase) /
   simplified (free-text gear, no model library) / defer.
3. **Stage as a first-class layer** — event → stages → advances, vs `stage` as a
   field on the advance (sheet is per-stage).
4. **Additions / Concerns / Pending** — structured per-advance fields (so they roll
   up to the summary report) vs the existing flags/comments mechanism.

Granular per-section field inclusion is confirmed **during Phase 4** as each section
is built (per the user's "ask as we go" guidance).
